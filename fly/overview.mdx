The `fly` CLI is the sole interface for controlling a Concourse server. Every operation — from pushing a pipeline configuration to streaming live build output to shelling into a running container — is performed through `fly`. Because `fly` communicates directly with the Concourse API, its binary version must always match the server version you are targeting; mismatched versions are reported as errors and fixed with a single `fly sync` command.

## Installing fly

fly binaries are distributed in two ways:

<CardGroup cols={2}>
  <Card title="Download from the Web UI" icon="browser">
    Open your Concourse instance in a browser. A download link for your operating system appears in the **bottom-right corner** of the screen. This always serves the binary that matches your server version.
  </Card>
  <Card title="GitHub Releases" icon="github">
    Download from the [Concourse GitHub releases page](https://github.com/concourse/concourse/releases). Pick the asset that matches your OS and architecture, make it executable, and place it on your `PATH`.
  </Card>
</CardGroup>

<Tip>
  After downloading or upgrading Concourse, run `fly -t <target> sync` to replace your local binary with the version served by the server. This is the fastest way to stay in sync.
</Tip>

## Global flags

These flags are available on every `fly` invocation and are defined on the top-level `FlyCommand` struct.

| Flag | Short | Description |
|---|---|---|
| `--target` | `-t` | Name of the saved Concourse target to use. Required by most commands. |
| `--version` | `-v` | Print the version of fly and exit. |
| `--verbose` | | Print raw API requests and responses for debugging. |
| `--print-table-headers` | | Print table headers even when output is redirected. |

```bash
# Print the installed fly version
fly --version

# Run a command with verbose API logging
fly --verbose -t ci pipelines
```

## Shell completion

`fly` can generate shell completion scripts for **bash**, **zsh**, and **fish**. Source the output in your shell profile to enable tab-completion for all commands and flags.

<CodeGroup>
```bash bash
fly completion --shell bash >> ~/.bashrc
source ~/.bashrc
```

```zsh zsh
fly completion --shell zsh >> ~/.zshrc
source ~/.zshrc
```

```fish fish
fly completion --shell fish > ~/.config/fish/completions/fly.fish
```
</CodeGroup>

## Raw API access with fly curl

The `fly curl` command (alias `c`) forwards HTTP requests directly to the Concourse API, using the stored credentials for the specified target. It is useful for exploring endpoints or scripting operations not yet covered by dedicated subcommands.

```bash
# Fetch server info
fly -t ci curl /api/v1/info

# List all pipelines via raw API
fly -t ci curl /api/v1/teams/main/pipelines
```

<Note>
  `fly curl` reuses the OAuth token from `~/.flyrc` for the selected target, so you do not need to pass credentials manually.
</Note>
