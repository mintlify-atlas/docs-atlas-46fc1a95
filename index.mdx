Concourse is an automation system written in Go that runs every task in an isolated container, enforces immutable build environments, and models your entire delivery pipeline in version-controlled YAML. It ships as a single binary and scales from a single Docker Compose node to a multi-worker production cluster.

## What Makes Concourse Different

Concourse pipelines are entirely **declarative YAML** — there is no GUI for building or editing pipelines. Every build runs in an **ephemeral container** spun up fresh for each task, and workers are **stateless** so they can be scaled, replaced, or lost without affecting builds. Resources are versioned artifacts (Git commits, Docker images, S3 objects) that trigger jobs and carry outputs between them.

## Get Up and Running

<Steps>
  <Step title="Start Concourse with Docker Compose">
    Download the sample compose file and start all services:
    ```bash
    wget https://concourse-ci.org/docker-compose.yml
    docker compose up -d
    ```
    Concourse is now running at `http://localhost:8080` with credentials `test` / `test`.
  </Step>
  <Step title="Download fly and log in">
    Download `fly` from the web UI or the releases page, then authenticate:
    ```bash
    fly -t ci login -c http://localhost:8080 -u test -p test
    ```
  </Step>
  <Step title="Set your first pipeline">
    Create a `hello-world.yml` pipeline file and apply it:
    ```bash
    fly -t ci set-pipeline -p hello-world -c hello-world.yml
    fly -t ci unpause-pipeline -p hello-world
    ```
  </Step>
  <Step title="Trigger a build and watch it run">
    ```bash
    fly -t ci trigger-job -j hello-world/say-hello --watch
    ```
  </Step>
</Steps>
